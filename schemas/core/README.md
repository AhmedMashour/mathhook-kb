## Schema Format

Each schema contains:

- **Core Content** (Required):
  - `topic`: Hierarchical identifier (e.g., `core.expressions`)
  - `title`: Human-readable title
  - `description`: Brief summary
  - `article.content`: Full markdown documentation preserved
  - `code_refs`: References to Rust/Python/Node.js implementations
  - `examples`: Multi-language code examples with explanations

- **Mathematical Content**:
  - `mathematical_definition`: LaTeX formulas and formal definitions
  - `performance`: Complexity, memory, timing characteristics
  - `use_cases`: Real-world applications

- **Metadata**:
  - `schema_version`: "1.0"
  - `source_file`: Original markdown file path
  - `generated_date`: "2025-12-15"
  - `related_topics`: Cross-references to other schemas
